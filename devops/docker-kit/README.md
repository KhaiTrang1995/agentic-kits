# Docker Kit

> AI Skill Kit for containerizing applications — Dockerfile, Compose, and CI pipelines, generated by an AI agent (Claude Code / any subagent-compatible IDE) following hardened production conventions.

> **Note — examples use .NET 10.**  
> Sample prompts and `_templates/Dockerfile.dotnet` target **.NET 10** (e.g. Minimal API). Skills are stack-agnostic: pass another runtime in the prompt when needed.

## Skills

| Skill | Description |
|-------|-------------|
| `/dockerfile` | Generate an optimized Dockerfile — multi-stage, non-root, Alpine-based |
| `/compose` | Generate docker-compose — multi-service, healthcheck, network segmentation |
| `/ci` | Generate a CI pipeline — build, test, scan, deploy |

## Rules

- `docker-conventions.md` — image, build, security, and registry conventions
- `approval-gate.md` — L1 (plan) / L2 (diff) / L3 (iterate) human checkpoints

## Templates

| Template | Description |
|----------|-------------|
| `Dockerfile.dotnet` | Multi-stage Dockerfile for **.NET 10** |
| `docker-compose.yml` | Multi-service compose with healthchecks |

## Usage

```bash
cp -r devops/docker-kit/.claude/ your-project/.claude/
cp -r devops/docker-kit/_templates/ your-project/_templates/

/dockerfile .NET 10 Minimal API --prod
/compose fullstack --with-db --with-redis
/ci .NET 10 --platform github --with-scan
```

## Design notes

- Every skill routes through the same 3 rules of `approval-gate.md` — the agent never runs `docker build`, `docker push`, or `docker-compose up` on your behalf. It only generates files; you review and run them.
- `docker-conventions.md` is the single source of truth for base images, security checklist, and `.env` handling — all 3 skills reference it instead of repeating the rules inline.
