# Docker Swarm Kit

> AI Skill Kit for Docker Swarm — stack files, service definitions, and secret/config management, generated by an AI agent following production-hardened conventions.

## Why a separate kit from docker-kit / k8s-kit

Docker Swarm sits between plain `docker-compose` (single host) and Kubernetes (full orchestration platform). It reuses the Compose file format but adds a `deploy:` stanza (replicas, placement, rolling update policy) and a distinct secret/config model (`docker secret`, `docker config` — mounted as files, not env vars). Those differences are enough that the conventions and skills need to be their own kit rather than a flag on `docker-kit`.

## Skills

| Skill | Description |
|-------|-------------|
| `/stack` | Generate a Swarm stack file (`docker-compose.yml` + `deploy:`) — replicas, placement, update/rollback policy |
| `/service` | Generate/update a single service definition — scaling, resource limits, health check |
| `/secret` | Generate Swarm secret/config management — external secrets, versioned configs, safe rotation |

## Rules

- `swarm-conventions.md` — deploy stanza, secrets/configs, networking, placement
- `approval-gate.md` — L1 (plan) / L2 (diff) / L3 (iterate) human checkpoints

## Templates

| Template | Description |
|----------|-------------|
| `docker-stack.yml` | Full stack file — services, deploy policy, secrets, overlay network |
| `traefik-proxy-stack.yml` | Reverse-proxy/ingress pattern for Swarm using Traefik labels (Swarm has no native Ingress resource) |

## Usage

```bash
cp -r devops/docker-swarm-kit/.claude/ your-project/.claude/
cp -r devops/docker-swarm-kit/_templates/ your-project/_templates/

/stack .NET API + Angular + Postgres --replicas 3
/service api --scale 5 --update-parallelism 2
/secret db-password --external
```

## Design notes

- Every generated stack file ships with `update_config` **and** `rollback_config` — a rolling update without a defined rollback is treated as incomplete output, not an optional extra.
- Secrets are always modeled as Swarm secrets (`docker secret create`, mounted at `/run/secrets/<name>`), never as plain environment variables — this is the main place `docker-kit`'s plain-Compose conventions do not carry over directly.
- The agent never runs `docker stack deploy`, `docker service update`, or `docker secret create` against a real Swarm — it only generates files/commands for you to review and run.
- If you need pod-level autoscaling, NetworkPolicy, or a real Ingress resource, that's `k8s-kit`, not this one — Swarm intentionally doesn't have those primitives.
