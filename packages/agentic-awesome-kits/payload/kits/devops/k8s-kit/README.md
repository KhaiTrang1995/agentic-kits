# K8s Kit

> AI Skill Kit for Kubernetes — manifests, Helm charts, and monitoring config, generated by an AI agent following production-hardened conventions.

## Skills

| Skill | Description |
|-------|-------------|
| `/deploy` | Generate K8s manifests — Deployment, Service, HPA, Ingress |
| `/helm` | Generate a Helm chart — values, templates, multi-env |
| `/monitor` | Generate monitoring config — Prometheus rules, Grafana dashboard |

## Rules

- `k8s-conventions.md` — labels, resources, probes, security context, networking
- `approval-gate.md` — L1 (plan) / L2 (diff) / L3 (iterate) human checkpoints

## Templates

| Template | Description |
|----------|-------------|
| `deployment.yaml` | Deployment with probes, resource limits, security context |
| `service.yaml` | ClusterIP service |
| `hpa.yaml` | HorizontalPodAutoscaler |
| `ingress.yaml` | Ingress + TLS via cert-manager |
| `helm-values.yaml` | Single source of truth `values.yaml` for `/helm` |
| `prometheus-rules.yaml` | Sample alert rules for `/monitor` |

## Usage

```bash
cp -r kits/devops/k8s-kit/.claude/ your-project/.claude/
cp -r kits/devops/k8s-kit/_templates/ your-project/_templates/

/deploy container-api --namespace prod --replicas 3 --with-ingress
/helm container-api
/monitor container-api --alerts --dashboard
```

## Design notes

- Resource limits, all 3 probes (liveness/readiness/startup), and a non-root `securityContext` are treated as mandatory, not optional — `/deploy` never emits a manifest missing them.
- `/helm --from-manifests <path>` converts manifests already generated by `/deploy` into a Helm chart instead of starting from scratch — the two skills are meant to be chained.
- The agent never runs `kubectl apply` or `helm install/upgrade` against a real cluster — it only generates files for you to review and apply.
